Syllabus2Calendar

Web app that turns pasted academic syllabi into calendars files using an LLM, with reminders and reports, instead of manual multi-event creation and academic organization

Technologies Used

  • Web: Vanilla JavaScript, HTML, and CSS: semester and class setup, syllabus paste/upload, API key and reminder/report settings
  • Proxy: Node.js: proxy to Flask API, static frontend hosting, and ICS file download
  • Backend: Python: Flask REST API, OpenAI (4o-mini) for parsing, icalendar for ICS generation. No database, processor state is per request; bulk ICS is written and returned

Impact and Results

  • Scope: Covers flow from pasted syllabi to ICS file: optional clarification, event extraction and date resolution, then calendar generation with timeline countdowns, configurable reminders, and pre-generated report content
  • Intelligent parsing: Two-step analysze-then-process flow with optional clarification for ambiguity; multi-format date handling and class-schedule integration (e.g. MWF, TTh); event classification (HW, EXA, PRO) for downstream timeline logic
  • Calendar behavior: Timeline countdown windows (e.g. 12-day exam countdown, project cycles, sequential homework windows); configurable per-day reminder windows and reminder count; pre-generated email report content with configurable report days and homework lookahead; one consolidated calendar from multiple syllabi
  • Output and footprint: Standard ICS for use in Google Calendar, Apple Calendar, Outlook, and other clients; no database or auth API key and config per session

Reader Suite

Desktop app for capturing and generating ebooks, plus a Progressive Web App (PWA) for reading offline, with optional backend for sync across devices

Technologies Used

  • Desktop: Python: GUI, automation, file generation, and encrypted custom format
  • Web: Vanilla JavaScript: PWA with offline storage (IndexedDB), Service Worker, and file handling
  • Backend: REST API (FastAPI), PostgreSQL, migrations, auth, and file storage

Impact and Results

  • Scope: Covers the full flow from capture to publish to read, plus cloud sync
  • User-focused design: Offline-first reading, no account required for local use, and data stays on device unless sync is used
  • Technical depth: Custom encrypted ebook format with key derivation and lazy decryption in the browser for security and performance improvements
  • Portable and deployable: Desktop app runs on Windows, macOS, and Linux; PWA works in major browsers and is installable; backend is standard REST and PostgreSQL.
