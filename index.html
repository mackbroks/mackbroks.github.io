Syllabus2Calendar

Web app that turns multiple academic course syllabi into organized calendar files using an LLM APIs, with reminders and reports; importable into Google Calendar, Apple Calendar, and Outlook

Tech stack

  • Web: Vanilla JavaScript, HTML, CSS: semester/class setup, syllabus paste/upload, API key and reminder/report settings
  • Proxy: Node.js: proxy to Flask API, static frontend hosting, ICS file download
  • Backend: Python, Flask REST API, OpenAI, icalendar: no database; per-request state; bulk ICS files written and returned

Situation

Students manually create hundreds of calendar events from syllabi to track assignments and due dates, often making errors, missing deadlines, and lack structured reminder systems

Task

Turn pasted syllabi into a single ICS calendar with configurable reminders, timeline countdowns, and pre-generated report content;without manual event entry

Action

Built a web app with syllabus paste/upload and an LLM-based two-step parse. Implemented multi-format date handling and class-schedule integration (e.g. MWF, TTh), event classification (HW, EXA, PRO), configurable reminder windows and report days, and one consolidated ICS from multiple syllabi. No database or authentication required

Result

End-to-end flow from syllabus to ICS; standard ICS works in Google Calendar, Apple Calendar, and Outlook; timeline countdowns (e.g. 12-day exam countdown), configurable reminders, and report content; single-session API key and config

Reader Suite

Desktop app for capturing and generating ebooks, plus a library suite for reading offline, with optional backend sync across devices

Tech stack

  • Desktop: Python: GUI, automation, file generation, and custom encrypted file format
  • Web:JavaScript: PWA with offline storage (IndexedDB), Service Worker, and file handling
  • Backend: REST API (FastAPI), PostgreSQL, migrations, auth, and file storage

Situation

Readers needed a way to capture content into ebooks, read offline on any device, and sync across devices without locking everything into a single third-party service

Task

Deliver a full flow from capture to read, with offline-first reading and optional cloud sync; keeping local use possible without an account.

Action

Built a Python desktop app for capture and generation of a custom encrypted ebook format with key derivation. Implemented a Progressive Web App (PWA) library and reader with IndexedDB and a Service Worker for offline reading. Added FastAPI backend with PostgreSQL, auth, and file storage for cross-device sync. Desktop runs on Windows, macOS, and Linux

Result

Single pipeline from capture to read; offline-first for local use; custom file encryption for security and performance; portable desktop and standard REST/PostgreSQL backend for deployability.
